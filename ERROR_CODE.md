# Error Codes (Frontend Reference)

백엔드 `ERROR_CODE.md`의 프론트 분기용 발췌본입니다. 코드는 `src/lib/api/errors.ts`의 `ErrorCode` 상수와 1:1.
모든 응답은 `{ success, data, code, message }`. 실패 시 `apiFetch`가 `ApiError(code, message, status)`를 던집니다.
`message`는 사용자에게 그대로 보여도 되는 한글입니다.

## AUTH — 인증/인가

| code | HTTP | 메시지 | 프론트 처리 |
|------|------|--------|-------------|
| `AUTH_001` | 401 | 인증이 필요합니다 | 로그인 페이지로 |
| `AUTH_002` | 401 | 토큰이 만료되었습니다 | apiFetch가 자동 갱신·재시도 |
| `AUTH_003` | 401 | 유효하지 않은 토큰입니다 | 토큰 폐기 → 로그인 |
| `AUTH_004` | 401 | 로그아웃된 토큰입니다 | 토큰 폐기 → 로그인 |
| `AUTH_005` | 403 | 접근 권한이 없습니다 | "권한 없음" 안내 |
| `AUTH_006` | 401 | 구글 인증에 실패하였습니다 | 로그인 재시도 |
| `AUTH_007` | 401 | Refresh Token이 유효하지 않습니다 | 토큰 폐기 → 로그인 |

## USER / FOLDER / FILE

| code | HTTP | 메시지 |
|------|------|--------|
| `USER_001` | 404 | 사용자를 찾을 수 없습니다 |
| `FOLDER_001` | 404 | 폴더를 찾을 수 없습니다 |
| `FOLDER_002` | 400 | 폴더를 자기 자신/하위로 이동할 수 없습니다 |
| `FOLDER_003` | 409 | 동일한 이름의 폴더가 이미 존재합니다 |
| `FOLDER_004` | 400 | 폴더 깊이 제한 초과 |
| `FILE_001` | 404 | 파일을 찾을 수 없습니다 |
| `FILE_002` | 400 | 지원하지 않는 파일 형식 (PDF, PNG, JPG만) |
| `FILE_003` | 400 | 파일 크기 제한 초과 |
| `FILE_004` | 400 | PDF 페이지 수 제한 초과 |
| `FILE_005` | 500 | 파일 업로드 실패 |
| `FILE_006` | 400 | 업로드 파일 수 제한 초과 |
| `FILE_007` | 400 | 저장 공간 제한 초과 |

## COMMON

| code | HTTP | 메시지 | 프론트 처리 |
|------|------|--------|-------------|
| `COMMON_001` | 400 | 잘못된 요청입니다 | 폼 유효성 에러 표시 |
| `COMMON_002` | 500 | 서버 내부 오류 | 일반 에러 토스트 |
| `COMMON_003` | 429 | 요청 횟수 제한 초과 | 잠시 후 재시도 |
| `COMMON_004` | 404 | 리소스를 찾을 수 없음 | 404 화면 |
| `COMMON_005` | 405 | 지원하지 않는 메서드 | (개발 오류) |

## 성공 코드 (참고)

| code | HTTP | 의미 |
|------|------|------|
| `COMMON_200` | 200 | 일반 처리 성공 |
| `COMMON_201` | 201 | 생성 성공 |
| `AUTH_200` | 200 | 로그인 성공 |
| `FILE_201` | 201 | 파일 업로드 성공 |
| `NOTE_202` | 202 | 노트 생성 요청 접수 (비동기) |
| `BLANK_202` | 202 | 빈칸 생성 요청 접수 (비동기) |
| `BLANK_200` | 200 | 채점 완료 |

> 성공 응답은 보통 `data`만 쓰면 되고 `code` 분기는 불필요합니다. `202`(비동기 접수)는 이후 status 폴링으로 이어집니다.
> 전체·최신 목록은 백엔드 `ERROR_CODE.md`가 기준입니다.
